🔄 Data Analysis Complete Process (Step-by-Step)

6-Step Framework with Real Example:

  1. 1. DEFINE (Think) - Understand business question
    • What problem to solve? (e.g., "Why did sales drop 15% in Q3?")
    • What metric matters? (Revenue, Units, Customer Count, Retention)
    • What's success? (Find root cause, improve by 10%)
  2. 2. COLLECT (Gather) - Get data from sources
    • Database query, CSV files, APIs, or systems
    • Time range, filters, relevant tables
    • Example: Extract 12 months of order data from database
  3. 3. CLEAN (Prepare) - Fix data quality issues
    • Remove duplicates, handle missing values (NULL, blanks)
    • Fix wrong data types, remove outliers
    • Standardize formats (dates, text case)
    • Example: Remove orders with NULL region_id, fix date formats
  4. 4. ANALYZE (Dig) - Find patterns & trends
    • Aggregate data (sum, count, average)
    • Compare groups (region, product, time period)
    • Calculate growth/decline, rank, percentages
    • Example: Group by month & region - identify West dropped 20%, East stable
  5. 5. VISUALIZE (Show) - Create charts & dashboards
    • Line chart for trends, bar for comparison, pie for composition
    • Interactive filters for drill-down exploration
    • Example: Line chart shows West declining from $500K to $400K
  6. 6. INSIGHT & RECOMMEND (Decide) - Explain & act
    • What's the root cause? (Lost sales rep, competitor, pricing)
    • What's the business impact? (Revenue loss, customer churn)
    • What should we do? (Action: Hire salespeople, run promotion)
    • Example: "West region lost top salesman Aug → hire 2 replacements + offer discount"

🏢 Real Business Example: Sales Performance Analysis

Problem: Q3 revenue dropped 15% vs Q2 (from $1M to $850K)

Step 2 - Collect: Extract from Sales DB: orders from Jan-Dec last year

Step 6 - Recommend: "Hire replacement for John NOW, investigate Product-C competitor pricing, run 10% discount in West for next 30 days"

🗄️ SQL - Syntax, Queries & Analysis Patterns

📌 Basic SELECT Query Structure

SELECT column1, column2, SUM(sales) AS total_sales
FROM orders
WHERE order_date >= '2024-01-01' AND status = 'completed'
GROUP BY column1, column2
HAVING SUM(sales) > 1000
ORDER BY total_sales DESC;

📌 SQL Functions for Analysis

Function Purpose Example
SUM() Total SELECT SUM(amount) FROM orders;
COUNT() Count non-NULL values SELECT COUNT(id) FROM orders; ← Count rows
COUNT(*) Count all rows (including NULL) SELECT COUNT(*) FROM orders;
AVG() Average SELECT AVG(amount) FROM orders;
MAX() / MIN() Highest / Lowest SELECT MAX(amount), MIN(amount) FROM orders;
GROUP BY Group rows, aggregate each group SELECT region, SUM(amount) FROM orders GROUP BY region;
HAVING Filter groups (like WHERE for GROUP BY) GROUP BY region HAVING SUM(amount) > 5000;

📌 JOIN Types - Connect Multiple Tables

JOIN Type Purpose SQL Example
INNER JOIN Only matching rows from both tables SELECT o.id, c.name FROM orders o INNER JOIN customers c ON o.customer_id = c.id;
LEFT JOIN All from left table + matching from right SELECT o.id, c.name FROM orders o LEFT JOIN customers c ON o.customer_id = c.id;
RIGHT JOIN All from right table + matching from left SELECT o.id, c.name FROM orders o RIGHT JOIN customers c ON o.customer_id = c.id;

📌 Real SQL Analysis Queries

-- 1. Total sales by region
SELECT region, SUM(amount) AS total_sales
FROM orders
WHERE status = 'completed'
GROUP BY region
ORDER BY total_sales DESC;

-- 2. Find top 10 customers by spending
SELECT customer_id, SUM(amount) AS total_spent
FROM orders
GROUP BY customer_id
ORDER BY total_spent DESC
LIMIT 10;

-- 3. Month-over-month growth (compare months)
SELECT 
  DATE_TRUNC('month', order_date) AS month,
  SUM(amount) AS monthly_revenue,
  LAG(SUM(amount)) OVER (ORDER BY DATE_TRUNC('month', order_date)) AS prev_month,
  ROUND(((SUM(amount) - LAG(SUM(amount)) OVER (ORDER BY DATE_TRUNC('month', order_date))) / LAG(SUM(amount)) OVER (ORDER BY DATE_TRUNC('month', order_date)) * 100), 2) AS growth_pct
FROM orders
GROUP BY DATE_TRUNC('month', order_date)
ORDER BY month;

-- 4. Find duplicates
SELECT id, COUNT(*) AS count
FROM orders
GROUP BY id
HAVING COUNT(*) > 1;

-- 5. Rank products by sales (with ROW_NUMBER)
SELECT product_id, SUM(amount) AS revenue,
  ROW_NUMBER() OVER (ORDER BY SUM(amount) DESC) AS rank
FROM orders
GROUP BY product_id;

💼 Interview Questions, Definitions & Explanations

📌 Key Definitions Every Data Analyst Should Know

  • KPI (Key Performance Indicator): Main metric to measure business success (e.g., Total Revenue, Customer Retention, Conversion Rate)
  • Root Cause Analysis: Finding the "why" behind a problem (e.g., "Sales dropped because we lost top salesman")
  • Cohort Analysis: Group customers by signup date, track behavior over time (e.g., Aug cohort has 60% retention vs Jul cohort 55%)
  • Retention Rate: % of customers staying (e.g., 100 Jan customers → 60 stayed in Feb = 60% retention)
  • Churn Rate: % of customers leaving (opposite of retention; if retention 60%, churn is 40%)
  • Aggregation: Combining detailed data into summary (e.g., sum sales by region)
  • Granularity: Level of detail in data (daily granularity vs monthly)
  • Outlier: Unusual data point that doesn't fit pattern (e.g., $1M sale when normal is $5K)
  • NULL / Missing Data: Empty cell with no value - must be handled in cleaning
  • Data Quality: Accuracy, completeness, consistency of data

📌 Common Interview Questions with Answers

Q1: "Tell me about your data analysis process."

Answer: I follow 6 steps: (1) Define the business question and success metric, (2) Collect data from relevant sources, (3) Clean data by removing duplicates and fixing types, (4) Analyze by grouping and aggregating, (5) Visualize findings in charts, (6) Recommend next steps based on insights. For example, if asked "Why did sales drop?", I'd extract 12 months of orders, check totals by region to identify where the drop happened, look for patterns like lost salespeople or competitor activity, then create a dashboard and recommend action.

Q2: "How do you clean messy data?"

Answer: First, I identify issues: (1) Missing values (NULLs) - I either remove rows or fill with average/mode, (2) Duplicates - check key fields and keep first occurrence, (3) Wrong types (date stored as text) - convert using to_datetime() or TEXT functions, (4) Inconsistent formats - use TRIM, PROPER, UPPER/LOWER for standardization, (5) Outliers - investigate if real or errors. I always document changes and validate the cleaned data.

Q3: "Compare VLOOKUP vs INDEX-MATCH."

Answer: VLOOKUP searches a table left-to-right, limited to forward direction. INDEX-MATCH is more flexible - it can search in any direction and combine with other functions. I prefer INDEX-MATCH because it's more robust. Example: =INDEX(Prices, MATCH(ProductID, Products, 0)) finds product in any order and returns price.

Q4: "Explain a SQL JOIN, which have you used?"

Answer: A JOIN connects rows from two tables based on matching keys. INNER JOIN returns only matching rows. LEFT JOIN keeps all rows from the left table + matching from right (good if you want every order even if customer not found). I use INNER JOIN when I want clean matches, LEFT JOIN when I need to keep all records. Example: LEFT JOIN Orders with Customers on customer_id to see all orders and their customer names (NULL if no match).

Q5: "Walk me through a real data analysis project."

Answer: Project: "Customer Churn Prediction". Business question: "Which customers are likely to leave?" Step 1: Defined churn as no purchase in 90 days. Step 2: Extracted customer purchase history + demographics. Step 3: Cleaned by removing test accounts, fixing dates. Step 4: Analyzed: found 30% inactive for 90+ days, new customers churn more (80%) vs old (10%). Step 5: Created dashboard showing churn risk by cohort. Step 6: Recommended: "Focus retention budget on customers aged 0-6 months; offer loyalty rewards." Result: Churn dropped 5% in next quarter.

Q6: "How do you validate your analysis is correct?"

Answer: I validate in multiple ways: (1) Spot-check numbers manually against source data, (2) Compare totals (sum of all regions should match overall total), (3) Check filters applied correctly (if filtering Q3 dates, verify all dates are Jul-Sep), (4) Ask business stakeholder "Does this number make sense?" (5) Look for logical consistency (can't have 120% growth), (6) Review data quality - how many NULLs did I exclude?

Q7: "What's a KPI and how do you set targets?"

Answer: KPI is a measurable metric tied to business goals. Examples: Revenue, Customer Acquisition Cost, Retention Rate, Conversion Rate. To set targets: (1) Look at historical trend (was it growing 5%/month?), (2) Compare to industry benchmark, (3) Set realistic goal (5-10% improvement is usually achievable), (4) Break into sub-targets (monthly milestones). Example: If current retention 70%, industry average 75%, set target 75% by end of Q2, track weekly.

Q8: "Tell me a time you found an error in data or analysis."

Answer: I was analyzing Q3 sales, found West region total $300K but my aggregate showed $350K. Root cause: I had duplicate order IDs from two database exports. I identified duplicates using GROUP BY id HAVING COUNT(*) > 1 in SQL, removed extras, recalculated. The corrected $300K matched source. This taught me to always check for duplicates in the first cleaning step. Now I verify row counts before and after cleaning.

📌 Technical Follow-Up Questions

  • "Why use GROUP BY instead of filtering?" → GROUP BY aggregates multiple rows into one; filtering just selects rows. GROUP BY needed for "total by region".
  • "What's the difference between COUNT(*) and COUNT(column)?" → COUNT(*) counts all rows; COUNT(column) skips NULL values. Use COUNT(*) for "how many rows?" and COUNT(column) for "how many have data?"
  • "When would you use HAVING vs WHERE?" → WHERE filters rows before grouping; HAVING filters groups after aggregation. Example: WHERE status='completed' filters orders first, then HAVING SUM(amount) > 1000 filters the summed results.
  • "Explain ROW_NUMBER() window function." → Ranks rows in order. Example: SELECT product, revenue, ROW_NUMBER() OVER (ORDER BY revenue DESC) ranks products by revenue. Used for "top 10" queries.

🧠 Memory Tips & Tricks for Fast Recall

📌 Acronyms & Mnemonics

  • DEFINE-COLLECT-CLEAN-ANALYZE-VISUALIZE-INSIGHT → (DCCAVI) - 6-step data analysis
  • SUM-AVERAGE-COUNT-MAX → (SACM) - Most common aggregations
  • VLOOKUP searches DOWN, INDEX-MATCH searches ANY direction → (DOWN vs ANY)
  • SELECT-FROM-WHERE-GROUP BY-ORDER BY → (SFWGO) - SQL query order
  • INNER-LEFT-RIGHT → (ILR) - 3 main JOIN types
  • Load-Transform-Model-Visualize-Publish → (LTMVP) - Power BI steps

📌 Quick Recall Formulas

  • Excel: IF always has 3 parts: condition, if-true-value, if-false-value
  • SQL: GROUP BY must include all non-aggregated columns
  • Python: Pandas chain: read → filter → group → aggregate → plot
  • Power BI: DAX measures use AGGREGATE functions: SUM, COUNT, AVG

📌 Visual Patterns to Remember

  • Trend/Time: Use LINE CHART
  • Comparison/Category: Use BAR CHART
  • Breakdown/Percentage: Use PIE CHART
  • Single Number: Use CARD
  • Many Details: Use TABLE

📌 Common Mistakes to Avoid

  • ❌ Forgetting to remove duplicates → Use COUNT(id) vs COUNT(DISTINCT id) to check
  • ❌ NULL values skewing averages → Always check ISNA().SUM() first
  • ❌ Time zones causing date mismatches → Standardize to UTC
  • ❌ Filtering but forgetting to apply → Always verify result count changed
  • ❌ Using wrong join and losing data → Verify row counts before/after JOIN
