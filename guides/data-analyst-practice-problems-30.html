Excel (10 Problems)

Focus: data cleaning, aggregation, and business insights.

Easy1) You have extra spaces in customer names. How do you clean them?
Answer: Use TRIM.
Explanation: TRIM removes leading/trailing and repeated spaces.
Easy2) Count completed orders in Status column.
Answer: =COUNTIF(StatusRange,"Completed")
Explanation: COUNTIF counts rows that match condition.
Easy3) Sum sales only for West region.
Answer: =SUMIF(RegionRange,"West",SalesRange)
Explanation: SUMIF adds values where one condition is true.
Medium4) Get CustomerName using CustomerID from another table.
Answer: =XLOOKUP(A2,CustomerIDs,CustomerNames,"Not found")
Explanation: XLOOKUP maps key to target value.
Medium5) Classify order value: High if sales > 1000 else Normal.
Answer: =IF(SalesCell>1000,"High","Normal")
Explanation: IF applies simple business rule.
Medium6) Build a monthly sales summary by region quickly.
Answer: Pivot Table with Month in Columns, Region in Rows, Sum of Sales in Values.
Explanation: Pivot is fastest summary tool in Excel.
Hard7) Find duplicate emails in a list.
Answer: Use =COUNTIF(EmailRange,EmailCell)>1 in helper column, then filter TRUE.
Explanation: Duplicates are values with count > 1.
Hard8) Calculate month-over-month growth %.
Answer: =(CurrentMonth-PreviousMonth)/PreviousMonth
Explanation: Measures change rate, not just absolute change.
Hard9) In one sheet you need lookup to left side (VLOOKUP cannot).
Answer: Use XLOOKUP or INDEX-MATCH.
Explanation: They support flexible lookup directions.
Hard10) Business asks: "Why revenue dropped in one region?" What Excel flow?
Answer: Pivot by month/region/product, compare periods, add contribution % table.
Explanation: Break drop into drivers: volume, price, mix.

SQL (10 Problems)

Focus: data cleaning checks, aggregation, and business insights.

Easy1) Show all rows from orders table.
Answer:
SELECT * FROM orders;
Explanation: Basic table read.
Easy2) Get completed orders only.
Answer:
SELECT * FROM orders WHERE order_status = 'completed';
Explanation: WHERE filters rows before any grouping.
Easy3) Count orders by region.
Answer:
SELECT region, COUNT(*) AS order_count
FROM orders
GROUP BY region;
Explanation: GROUP BY builds category totals.
Medium4) Total revenue by country, highest first.
Answer:
SELECT country, SUM(amount) AS revenue
FROM orders
GROUP BY country
ORDER BY revenue DESC;
Explanation: Aggregate then sort for ranking.
Medium5) List customers with no orders.
Answer:
SELECT c.customer_id, c.customer_name
FROM customers c
LEFT JOIN orders o ON c.customer_id = o.customer_id
WHERE o.customer_id IS NULL;
Explanation: LEFT JOIN keeps all customers, null match means no order.
Medium6) Filter grouped results where revenue > 10000.
Answer:
SELECT region, SUM(amount) AS revenue
FROM orders
GROUP BY region
HAVING SUM(amount) > 10000;
Explanation: HAVING filters after aggregation.
Hard7) Find duplicate emails.
Answer:
SELECT email, COUNT(*) AS c
FROM customers
GROUP BY email
HAVING COUNT(*) > 1;
Explanation: Data quality check.
Hard8) Top 3 customers by revenue per region.
Answer:
WITH rev AS (
  SELECT region, customer_id, SUM(amount) AS revenue
  FROM orders
  GROUP BY region, customer_id
)
SELECT *
FROM (
  SELECT rev.*, ROW_NUMBER() OVER(PARTITION BY region ORDER BY revenue DESC) rn
  FROM rev
) x
WHERE rn <= 3;
Explanation: Aggregate first, rank inside region, keep top-N.
Hard9) Running monthly revenue trend.
Answer:
WITH monthly AS (
  SELECT DATE_TRUNC('month', order_date) m, SUM(amount) revenue
  FROM orders
  GROUP BY 1
)
SELECT m, revenue,
       SUM(revenue) OVER (ORDER BY m) AS running_revenue
FROM monthly;
Explanation: Window SUM creates cumulative KPI.
Hard10) Business asks: "Which channel caused revenue drop?"
Answer: Compare channel revenue for current vs previous period with grouped query and % change.
Explanation: Insight comes from period comparison by segment.

Python (10 Problems)

Focus: data cleaning, aggregation, and business insights using pandas.

Easy1) Load CSV file.
Answer:
import pandas as pd
df = pd.read_csv("sales.csv")
Explanation: First step in pandas workflow.
Easy2) Remove duplicate rows.
Answer:
df = df.drop_duplicates()
Explanation: Avoid double counting.
Easy3) Check missing values in each column.
Answer:
df.isna().sum()
Explanation: Quick data quality summary.
Medium4) Convert order_date to datetime safely.
Answer:
df["order_date"] = pd.to_datetime(df["order_date"], errors="coerce")
Explanation: Invalid dates become null for cleanup.
Medium5) Filter only West region records.
Answer:
west = df[df["region"] == "West"]
Explanation: Boolean filter by condition.
Medium6) Revenue by region summary.
Answer:
summary = df.groupby("region", as_index=False)["sales"].sum()
Explanation: groupby + sum for KPI table.
Hard7) Merge orders with customers by customer_id.
Answer:
merged = orders.merge(customers, on="customer_id", how="left")
Explanation: Same concept as SQL LEFT JOIN.
Hard8) Create monthly revenue trend and plot.
Answer:
df["month"] = df["order_date"].dt.to_period("M").astype(str)
m = df.groupby("month", as_index=False)["sales"].sum()
m.plot(x="month", y="sales", kind="line")
Explanation: Convert date -> month, then aggregate and visualize.
Hard9) Find top 5 customers by revenue.
Answer:
top5 = (df.groupby("customer_id", as_index=False)["sales"].sum()
          .sort_values("sales", ascending=False)
          .head(5))
Explanation: Aggregate, sort, then slice.
Hard10) Business asks: "Why did retention fall?" What Python flow?
Answer: Define cohort month, compute active users by month offset, compare retained vs churned behavior.
Explanation: Product insight needs cohort and trend analysis, not one raw count.
