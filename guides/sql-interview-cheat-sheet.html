Patterns

Fast Query Templates

Top 3 Products Per Region

WITH product_rev AS (
  SELECT region, product_id, SUM(net_revenue) AS revenue
  FROM sales
  GROUP BY 1,2
)
SELECT *
FROM (
  SELECT *, ROW_NUMBER() OVER (PARTITION BY region ORDER BY revenue DESC) AS rn
  FROM product_rev
) x
WHERE rn <= 3;

Month over Month Revenue

WITH m AS (
  SELECT DATE_TRUNC('month', order_date) AS month, SUM(amount) AS revenue
  FROM orders
  GROUP BY 1
)
SELECT month, revenue,
       revenue - LAG(revenue) OVER (ORDER BY month) AS mom_change
FROM m;

Duplicate Removal

WITH d AS (
  SELECT *, ROW_NUMBER() OVER (PARTITION BY transaction_id ORDER BY updated_at DESC) AS rn
  FROM transactions
)
SELECT * FROM d WHERE rn = 1;
BI Integration

Excel and Power BI Interview Notes

  • For Excel outputs, produce one clean row per business grain and avoid duplicate joins.
  • For Power BI, keep fact and dimension tables separate and stable key relationships.
  • For Tableau performance, pre-filter large data and avoid high-cardinality detail in dashboard sources.
Edge Cases

Final 30-Second Checklist

  • Did NULLs change your arithmetic?
  • Did join grain multiply rows?
  • Did ORDER BY for windows match business time logic?
  • Did ties in rank functions impact filtering?
  • Does your output table match dashboard grain?
