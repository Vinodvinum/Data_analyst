How to Use This Guide

If you are preparing seriously, do not read everything passively. For each section, write the answer first, then compare it to the sample approach here. That is how you build speed and clarity.

Recommended study order: SQL first, then Python, then business cases, then Power BI and Excel. That order matches how many interviews actually test candidates.
Good practice datasets to use while studying:
  • Sales tables: orders, customers, products, regions.
  • Finance tables: invoices, payments, budgets.
  • User tables: signup date, churn flag, activity events, segments.
Foundation

1. Foundation Mindset

What interviewers want

  • Can you solve the problem correctly?
  • Can you explain your logic clearly?
  • Do you notice edge cases?
  • Can you connect the result to a business decision?

Answer structure

  • Restate the problem.
  • Define the data grain and assumptions.
  • Show the method or query.
  • Explain the result and business meaning.
Common pattern: the candidate who explains tradeoffs and edge cases usually performs better than the candidate who only gives the final answer.
Core section

2. SQL Interview Preparation

SQL is the most important technical area for data analyst interviews. You need to be comfortable with filtering, grouping, joins, subqueries, CTEs, and window functions, and you must be able to explain why a query works.

2.1 Basic to Advanced Questions

SELECT, WHERE, GROUP BY, HAVING

Interview point: WHERE filters rows before grouping, HAVING filters groups after aggregation.

SELECT country, COUNT(*) AS order_count
FROM orders
WHERE order_status = 'completed'
GROUP BY country
HAVING COUNT(*) >= 5;

Business use case: find countries with enough order volume to justify a regional campaign.

Joins and edge cases

Know inner, left, full, and self joins. Explain row loss, duplicate expansion, and null preservation.

SELECT c.customer_id, c.customer_name, o.order_id
FROM customers c
LEFT JOIN orders o
  ON c.customer_id = o.customer_id;

Business use case: identify customers with no orders after onboarding.

2.2 Hard SQL Questions

Question: Find the top 3 customers by revenue in each country.

Solution:

WITH customer_revenue AS (
    SELECT
        c.country,
        o.customer_id,
        SUM(o.quantity * o.unit_price - o.discount_amount) AS revenue
    FROM orders o
    JOIN customers c ON o.customer_id = c.customer_id
    WHERE o.order_status = 'completed'
    GROUP BY c.country, o.customer_id
), ranked AS (
    SELECT *,
           ROW_NUMBER() OVER (
               PARTITION BY country
               ORDER BY revenue DESC
           ) AS rn
    FROM customer_revenue
)
SELECT country, customer_id, revenue
FROM ranked
WHERE rn <= 3
ORDER BY country, revenue DESC;

Explanation: aggregate first, rank within each country, then keep only the top rows.

Question: Find duplicate customer emails.

Solution:

SELECT email, COUNT(*) AS duplicate_count
FROM customers
GROUP BY email
HAVING COUNT(*) > 1;

Explanation: duplicates are detected by grouping on the field and checking counts greater than one.

Question: Calculate running revenue by month.

Solution:

WITH monthly AS (
    SELECT
        DATE_TRUNC('month', order_date) AS month,
        SUM(quantity * unit_price - discount_amount) AS revenue
    FROM orders
    WHERE order_status = 'completed'
    GROUP BY 1
)
SELECT
    month,
    revenue,
    SUM(revenue) OVER (ORDER BY month) AS running_revenue
FROM monthly
ORDER BY month;

Explanation: first calculate monthly totals, then use a window function to accumulate them over time.

Question: Build a cohort retention query.

Solution idea: define cohort month from the first purchase, calculate active months after that, and measure retention by cohort and month offset.

WITH first_purchase AS (
    SELECT customer_id, MIN(order_date) AS first_order_date
    FROM orders
    GROUP BY customer_id
), activity AS (
    SELECT
        o.customer_id,
        DATE_TRUNC('month', f.first_order_date) AS cohort_month,
        DATE_TRUNC('month', o.order_date) AS activity_month
    FROM orders o
    JOIN first_purchase f ON o.customer_id = f.customer_id
)
SELECT cohort_month, activity_month, COUNT(DISTINCT customer_id) AS active_customers
FROM activity
GROUP BY cohort_month, activity_month;

Explanation: interviewers want to see that you can think in terms of cohorts, time buckets, and distinct customers.

Python

3. Python Interview Preparation

In interviews, Python is usually tested through pandas tasks, data cleaning, joins, and compact analysis. You do not need to prove you are a software engineer. You need to show you can reason about data clearly and efficiently.

Cleaning a messy table

import pandas as pd

sales = pd.read_csv("sales.csv")
sales.columns = sales.columns.str.strip().str.lower()
sales = sales.drop_duplicates()
sales["order_date"] = pd.to_datetime(sales["order_date"], errors="coerce")
sales["sales"] = pd.to_numeric(sales["sales"], errors="coerce")
sales["region"] = sales["region"].fillna("Unknown")

Interview explanation: clean column names first, fix types early, then handle missing values and duplicates.

Groupby and merging

customer_revenue = (
    sales.groupby("customer_id", as_index=False)["sales"]
    .sum()
    .rename(columns={"sales": "revenue"})
)

merged = customer_revenue.merge(customers, on="customer_id", how="left")

Business use case: identify the highest-value customers and enrich them with region and segment data.

Practical task: remove missing dates, convert columns, and create a segment label. 
sales["segment"] = pd.cut(
    sales["sales"],
    bins=[0, 100, 500, float("inf")],
    labels=["Low", "Medium", "High"],
    include_lowest=True
)

Explanation: use clear buckets when the business needs simple categories for reporting.

Performance tip: use vectorized operations and avoid row-by-row loops unless there is no alternative. In interviews, say why vectorized code is faster and easier to maintain.
Power BI

4. Power BI and Dashboard Interview

DAX questions

  • Explain CALCULATE and why it is powerful.
  • Describe row context vs filter context.
  • How do you calculate year-over-year growth?
YoY Growth % =
DIVIDE(
    [Sales Amount] - CALCULATE([Sales Amount], SAMEPERIODLASTYEAR('Date'[Date])),
    CALCULATE([Sales Amount], SAMEPERIODLASTYEAR('Date'[Date])),
    0
)

Data modeling scenarios

  • Why is star schema usually preferred?
  • When would a many-to-many relationship be needed?
  • How do you secure data with RLS?

Strong answer: start from the business grain, then explain fact tables, dimensions, relationships, and filter direction.

Dashboard interview trap: candidates often talk only about visuals. Top answers explain data model, measures, audience, and business action together.

6. Business Case Questions

These questions matter because they show how you think when the answer is not in the data alone.

Revenue drop analysis

How to think: compare current period vs previous periods, then segment by product, region, channel, and customer type.

What to ask: did volume drop, price drop, conversion drop, or mix shift?

Step-by-step: validate the metric, isolate time period, segment the drop, identify drivers, and propose action.

Customer churn investigation

How to think: define churn clearly first. Different companies define it differently.

What to ask: who is churned, when do they count as churned, and is this voluntary or involuntary?

Step-by-step: define churn, build cohort view, compare behavior of retained vs churned users, and identify leading indicators.

Product performance analysis

How to think: do not only look at total sales. Check units, margin, repeat purchases, return rate, and seasonality.

What to ask: is the product underperforming because of traffic, conversion, pricing, or stock availability?

Step-by-step: examine trend, compare against peers, look at margin, and test whether the product belongs in the portfolio.

Hard level

7. Product and Analytical Thinking

Metrics design

Good analysts define metrics so that the business can act on them. A KPI should be clear, stable, and tied to a decision.

MetricQuestion it answers
Conversion rateHow many users completed the desired action?
RetentionDo users come back after first use?
Average order valueHow much does each order contribute?

A/B testing and funnels

  • State the hypothesis clearly.
  • Explain the primary metric.
  • Check sample size and bias.
  • Analyze funnel drop-off step by step.

Business use case: an onboarding redesign should reduce funnel drop-off from signup to activation.

Common traps

10. Common Mistakes and Traps

SQL mistakes

  • Confusing WHERE and HAVING.
  • Forgetting joins can multiply rows.
  • Using COUNT(*) when COUNT(column) was intended.
  • Ignoring null handling.

Business mistakes

  • Jumping to conclusions without checking definitions.
  • Not asking which metric the business cares about.
  • Not separating volume, price, and mix effects.
  • Not stating assumptions clearly.
AI

11. AI for Interview Prep

AI is useful for practice, but you still need to verify the logic and improve the explanation yourself.

Simple way to use AI: ask it to quiz you one question at a time, then explain your answer back in your own words. That forces active recall instead of passive reading.

Real-data prompt

Use a sales table with order date, country, product, and revenue. Ask the AI to generate a revenue-drop diagnostic plan, then compare it to the numbers yourself.

Practice prompt

Act as a senior data analyst interviewer.
Ask me one SQL question, one Python question, and one business case question.
Wait for my answer before scoring me and giving feedback.
Prompt example 1:
"Act as a senior data analyst interviewer. Ask me 5 medium SQL questions and wait for my answer before giving feedback."

Prompt example 2:
"Review this DAX measure and tell me whether the filter context is correct. Also suggest a simpler version if possible."

Prompt example 3:
"Give me a mock interview for a product analytics role with SQL, Python, Excel, and business questions. Include scoring criteria."
Important: never copy an answer blindly from AI in an interview. Use it to practice structure, then rewrite the answer in your own words.
Bonus

12. Top Company Level Preparation

What top companies test

  • Can you solve messy, ambiguous problems?
  • Can you explain tradeoffs clearly?
  • Can you think about users and business impact?
  • Can you stay calm when a question has edge cases?

How to stand out

  • State assumptions before writing the query.
  • Check edge cases explicitly.
  • Explain why your solution is robust.
  • Connect every answer to a decision or KPI.

If you can solve SQL correctly, explain business impact, and communicate clearly, you will already outperform many candidates.
