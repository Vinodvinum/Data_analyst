1. Data Analysis Process (Step-by-Step)

  1. Define: understand business question and KPI.
  2. Collect: gather data from files, DBs, or tools.
  3. Clean: fix missing values, duplicates, wrong types.
  4. Analyze: aggregate, compare trends, find patterns.
  5. Visualize: show metrics in charts/tables.
  6. Insight: explain what happened and what to do next.

Real business case: Sales dropped 15% in West region. You define "drop" as month-over-month revenue change, collect orders data, clean missing region values, analyze by product/channel, visualize trend by month, and finally recommend discount correction + stock fix for underperforming products.

2. Excel for Data Analysis

Important formulas

  • SUM, AVERAGE, COUNT, COUNTA
  • IF, IFS, AND, OR
  • VLOOKUP, XLOOKUP, INDEX-MATCH
  • COUNTIF, SUMIF, SUMIFS
=SUM(E2:E100)
=IF(F2>1000,"High","Low")
=XLOOKUP(A2, IDs, Names, "Not found")

Data cleaning functions

  • TRIM: remove extra spaces
  • PROPER / UPPER / LOWER: fix case
  • CLEAN: remove non-printable chars
  • TEXT, VALUE: format and type correction
=TRIM(A2)
=PROPER(B2)
=CLEAN(C2)

Pivot Table in simple steps

  1. Select data table.
  2. Insert -> PivotTable.
  3. Drag category to Rows, metrics to Values.
  4. Add Date to Columns for trend view.
  5. Use filters/slicers for region/product.

Example solved: Sales data with columns Date, Region, Product, Amount. Pivot by Region and sum Amount -> West is lowest. Add Product under Region -> identify low products driving decline.

3. SQL for Data Analysis

Basic syntax

SELECT region, SUM(amount) AS revenue
FROM orders
WHERE order_status = 'completed'
GROUP BY region
ORDER BY revenue DESC;

Joins

  • INNER JOIN: only matching rows
  • LEFT JOIN: all left rows + matched right rows
  • RIGHT JOIN: all right rows + matched left rows
  • FULL JOIN: all rows from both sides

Aggregations

SELECT country, COUNT(*) AS orders, AVG(amount) AS avg_order, SUM(amount) AS total
FROM orders
GROUP BY country;

Real-world query: find customers with no orders.

SELECT c.customer_id, c.customer_name
FROM customers c
LEFT JOIN orders o ON c.customer_id = o.customer_id
WHERE o.customer_id IS NULL;

4. Python for Data Analysis

Use pandas for table analysis, numpy for numeric operations, matplotlib for charts.

import pandas as pd
import numpy as np
import matplotlib.pyplot as plt

# Load
df = pd.read_csv("datasets/python-sales.csv")

# Clean
df = df.drop_duplicates()
df["order_date"] = pd.to_datetime(df["order_date"], errors="coerce")
df["sales"] = pd.to_numeric(df["sales"], errors="coerce")
df["region"] = df["region"].fillna("Unknown")

# Filter
west = df[df["region"] == "West"]

# Group
summary = df.groupby("region", as_index=False)["sales"].sum()

# Visualize
summary.plot(kind="bar", x="region", y="sales", legend=False)
plt.title("Sales by Region")
plt.show()

Step-by-step meaning: load -> clean -> filter -> group -> chart -> explain business impact.

5. Power BI

  1. Load: import CSV/DB tables.
  2. Transform: clean in Power Query.
  3. Model: create fact-dimension relationships.
  4. Visualize: add KPI cards, trend charts, slicers.

Important DAX

Sales Amount = SUM(Sales[Amount])
Order Count = COUNTROWS(Sales)
Average Order Value = DIVIDE([Sales Amount], [Order Count], 0)
Sales YTD = TOTALYTD([Sales Amount], 'Date'[Date])

Dashboard example: one page with total sales, MoM trend, top products, region filter, and under-target alert card.

7. Interview Preparation

Top 20 common interview questions (short answers)

  1. What is NULL? Missing/unknown value.
  2. Difference between WHERE and HAVING? WHERE before grouping, HAVING after.
  3. What is JOIN? Combining tables by key.
  4. What is KPI? Measurable business target metric.
  5. What is data cleaning? Fixing errors, nulls, duplicates, and types.
  6. When use LEFT JOIN? Keep all rows from primary table.
  7. What is GROUP BY? Aggregate by category.
  8. What is star schema? Fact table + dimension tables model.
  9. What is DAX measure? Dynamic calculation under filter context.
  10. What is filter context? Active filters affecting calculation.
  11. Why use pivot tables? Quick summaries without coding.
  12. What is window function? Row-level analytics over partitions.
  13. What is cohort analysis? Group users by start period and track behavior.
  14. Why data validation? Ensure data quality before analysis.
  15. Difference between mean and median? Mean average, median middle value.
  16. What is outlier? Extreme value outside normal range.
  17. Why use Python if SQL exists? Advanced cleaning and modeling flexibility.
  18. Why use Power BI? Interactive dashboarding and business storytelling.
  19. What is ETL? Extract, Transform, Load pipeline.
  20. How to explain an insight? Metric + change + driver + action.

8. Memory Tips & Tricks

  • Excel: IF = condition question, LOOKUP = map value, IF+SUM = conditional KPI.
  • SQL flow: SELECT -> FROM -> WHERE -> GROUP BY -> HAVING -> ORDER BY.
  • Python flow: load -> inspect -> clean -> transform -> aggregate -> visualize.
  • Power BI flow: load -> transform -> model -> measure -> dashboard.
  • Learning hack: every concept = one small task + one explanation in your own words.
