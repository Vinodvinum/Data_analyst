Important Definitions

Fact Table

Transaction table with numeric metrics at a defined grain, such as orders or invoices.

Dimension Table

Descriptive table used for slicing facts, such as customer, date, product, region.

Star Schema

Model design where fact table is central and dimensions connect around it.

Filter Context

Current filters applied by slicers, visuals, or DAX logic.

Cohort

Group of users/customers who started in the same period.

Retention

Percentage of users returning after initial acquisition period.

Tricks and Memory Shortcuts

  • SQL solving order: define grain -> filter -> join -> aggregate -> rank -> present result.
  • DAX solving order: write base measure first, then wrap with CALCULATE logic.
  • Excel cleanup order: fix headers -> types -> blanks -> duplicates -> formulas.
  • Python cleanup order: inspect data -> clean nulls -> types -> joins -> aggregate.
  • Dashboard rule: one page should answer one business story, not every question.
